By using the pre-built assets of Elements, we avoid building and re-building basic elements and can spend more time and effort customizing our digital products to address customer needs. Our goal with this portal is to provide a central reference point for our established design patterns, user interface rules, and design guidelines for the Elements Design System.

These guidelines will not answer every question you have. While our goal is to address commonly-used design pattern or components, there will be unique situations that may not be directly addressed here.

If you run into something that you feel is missing, contact the CX & Design Council.
Leverage our Design Principles when stuck between 2 design decision points or looking for guidance.

Over time, with fidelity to the Elements Design System, Sanofi Digital ecosystem will start to resemble a family of applications, with a familiar and consistent user experience.

### What Does It Include?

Elements is a library. To use it, you must have a React application that can consume JavaScript modules. We recommend Typescript, as it allows for a much better experience.

- The design kit is built and maintained in Figma. It includes a set of foundational components and a reference architecture.
- The component library hosted on Github mirrors the Figma components and is built using Typescript, React and CSS Modules.
- Storybook acts as both the developer sandbox and the documentation platform.
- Github actions run all forms of testing builds and deploys Storybook as a static site.

### Definitions

- Design Kit — a library of shared styles, symbols or components that can be used by product teams to design or prototype new experiences.
- Design Tokens — the overall visual design of a digital product. This foundation defines characteristics such as typography, colors, icons, spacing and information architecture.
- Component Library — a set of JavaScript components that are version controlled and are composed to build one or more products.
- Documentation — a set of guidelines on how to consume the Design System, design and dev considerations and detailed documentation for each component. The documentation site often includes a live playground that is aimed at the consumers to try out components in the browser.
- Developer Sandbox — a tool for developing components in isolation, document use cases and write structural or visual tests.
