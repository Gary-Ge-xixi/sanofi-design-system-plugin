结论快照

最适合的角色Ant Design 适合快速做企业后台 React 页面；shadcn 适合把组件源码纳入产品工程并长期自定义；本地 Sanofi Elements 适合把品牌设计规范、组件证据、页面规则和 HTML 验证串成交付闭环。
交付给研发Ant Design 和 shadcn 的理想交付物是 React/Tailwind 源码，不是纯 HTML。本地链路的目标就是静态 HTML，并带有 `data-primitive`、`data-pattern`、`data-source-doc-id`、`data-token-source` 等追踪信息。
规范一致性Ant Design 默认会输出 Ant 体系观感，shadcn 默认会输出中性 Tailwind/Radix 观感；两者都需要二次品牌化。本地输出直接绑定 Sanofi Elements token、规则和 Storybook/文档来源。
当前短板本地链路最重，前期要维护 registry、contract、template、validator；同时 Table 在该样本中仍是明确标注的 fallback，不应冒充官方 Table primitive。本次 1280 宽截图还显示本地表格需要横向滚动才能看到右侧数量列，说明还应补视觉截图门禁。
